4. Key Insights

Where Novada Wins

  • Structured field extraction: 87.5% vs Firecrawl 75%, Tavily 75%. The fields param works — all 7 extract-based tasks hit 100% field coverage.
  • Useful chars ratio = 100%. Every successful extraction scored quality ≥3/5. No nav-pollution, no cookie banners. AI agents get signal, not noise.
  • Static/cached content is sub-1s. MDN 192ms, Python docs 203ms, TechCrunch 226ms, BBC 318ms, Smashing Mag 237ms.
  • Cost: 4× cheaper than Firecrawl, 5× cheaper than Tavily at $1/1k.

Where Novada Is Still Weak

  • P95 tail latency. JS-heavy / anti-bot sites push to 17–43s. GitHub/trending = 44s, Goodreads = 33s, quotes.toscrape = 10–19s. Firecrawl handles these in 1–2s via fire-engine.
  • Medium.com timeout (60s). JS paywall. novada_extract escalation ceiling too high — should cap at 30s and return partial.
  • StackOverflow content too thin (642 chars). Login wall / paginated. Need auth-aware detection.
  • Amazon novada_scrape = 0 chars. API-level failure. Separate from extract tool.
  • example.com = 15.8s (should be <100ms). Unexpected JS escalation on a static page.
