FAQ 常见问题

Everything you need to know about Novada MCP — clients, keys, data, limits, and open source. 关于 Novada MCP 你需要了解的一切 — 客户端、密钥、数据、配额与开源。

How is this different from calling the Novada REST API directly? 这和直接调用 Novada REST API 有什么区别?

The MCP endpoint exposes Novada tools through the Model Context Protocol so any MCP-compatible agent (Claude, Cursor, Cline, Codex…) can discover and call them with a structured contract — no client SDK, no per-tool wrapper code. Underneath, it dispatches to the same Novada backend. MCP 端点通过 Model Context Protocol 暴露 Novada 工具,任何兼容 MCP 的 Agent(Claude、Cursor、Cline、Codex 等)都能以结构化协议自动发现与调用 — 无需客户端 SDK,无需逐工具封装代码。底层仍由 Novada 后端处理。

Do I need a separate key for MCP? MCP 需要单独的密钥吗?

No. The same Novada API key works for the hosted MCP, the local npx novada-mcp, and the REST API. Quota counters are per token. 不需要。同一把 Novada API Key 同时适用于托管 MCP、本地 npx novada-mcp 与 REST API。额度按 Token 计算。

Why can't I use browser tools on the hosted endpoint? 为什么托管端无法使用浏览器工具?

Headless browsers (Playwright) don't run inside serverless edge functions. The hosted endpoint returns 501 with an agent_instruction telling the agent to switch to the local MCP via npx novada-mcp — same key, same tool names. 无头浏览器(Playwright)无法在 serverless 边缘函数中运行。托管端会返回 501 并附带 agent_instruction,引导 Agent 切换到本地 MCP npx novada-mcp — 同一密钥、同一工具名。

Can I send the key as a header instead of a URL query? 能否用请求头代替 URL 查询参数传递密钥?

Yes. Authorization: Bearer YOUR_API_KEY against https://mcp.novada.com/mcp works identically. Prefer the header when sharing screens or storing client configs in repos. 可以。对 https://mcp.novada.com/mcp 使用 Authorization: Bearer YOUR_API_KEY 效果相同。分享屏幕或将客户端配置纳入仓库时,推荐使用请求头方式。

What's the difference between hosted and npx novada-mcp? 托管版与 npx novada-mcp 有何不同?

Hosted = zero install, paste-a-URL, always latest, but no browser tools. Local npx novada-mcp = runs everything including Playwright browser flows, works offline-from-cloud, but you manage updates yourself. 托管版 = 零安装、粘贴 URL、始终最新,但不支持浏览器工具。本地 npx novada-mcp = 全功能含 Playwright 浏览器流程、可离线使用,但需自行管理更新。

Which MCP clients are supported? 支持哪些 MCP 客户端?

Claude Desktop, Claude Code, Cursor, Codex, Cline, and Windsurf are all supported. See the Install page for per-client config snippets you can copy and paste. 支持 Claude Desktop、Claude Code、Cursor、Codex、Cline 与 Windsurf。请前往接入页面查看各客户端的可复制配置片段。

Where is my data processed and where does it run? 我的数据在哪里处理?服务运行在哪里?

The hosted MCP server runs on Vercel Edge (global PoPs, auto-SSL). Each request dispatches to Novada's backend for tool execution. Your API key is used only as a call token — it is not stored beyond the request lifetime. No request payloads are logged by the MCP layer. 托管 MCP 服务器运行于 Vercel Edge(全球节点,自动 SSL)。每次请求均分发至 Novada 后端执行工具。你的 API 密钥仅用作调用凭证 — 请求完成后不会保留。MCP 层不会记录请求内容。

What happens when I hit the free quota? 免费额度用完后会怎样?

The server returns HTTP 429. Your quota resets monthly per token. To get a higher limit, visit the Novada dashboard and upgrade your plan. 服务器将返回 HTTP 429。额度按 Token 每月重置。如需更高配额,请前往 Novada 控制台 升级套餐。

Is Novada MCP open source? Novada MCP 是开源的吗?

Yes. The hosted server repository is public at github.com/NovadaLabs/novada-mcpserver. The underlying tool implementations come from the novada-mcp npm package. 是的。托管服务器仓库已公开于 github.com/NovadaLabs/novada-mcpserver。底层工具实现来自 novada-mcp npm 包。
