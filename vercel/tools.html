▸ SEARCH & EXTRACT ×2

Search & Extract搜索与提取

Find pages and read them. Use novada_search to find relevant URLs across 4 engines; use novada_extract to read clean content from a URL you already have. 找到页面并读取内容。用 novada_search 跨 4 个引擎找到相关 URL；用 novada_extract 从已知 URL 读取干净内容。

novada_search
RARE

Search the web via Google, Bing, DuckDuckGo, or Yandex. Use when you need to find relevant pages but don't know the URL. Returns titles, URLs, and snippets. For full page content, follow up with extract. 通过 Google、Bing、DuckDuckGo 或 Yandex 搜索网页。需要找相关页面但不知道 URL 时使用。返回标题、URL 和摘要。需要整页内容时再用 extract。

Use when you need to find relevant pages from the open web. 使用场景：需要从公开网络找到相关页面。
novada_extract
EPIC

Read clean content from one or more URLs. Use when you have a specific page URL and need its content. Handles anti-bot protection automatically. For raw HTML, use format='html'. For multiple pages on one site, use crawl instead. 从一个或多个 URL 读取干净内容。已有具体页面 URL 并需要其内容时使用。自动处理反爬保护。需要原始 HTML 用 format='html'。同一站点多页请改用 crawl。

Use when you have a URL and need its clean content. 使用场景：已有 URL 并需要其干净内容。
▸ PLATFORM SCRAPE ×1

Platform Scraper平台抓取

Structured records from known platforms — not general web pages. For arbitrary URLs use novada_extract instead. 从已知平台获取结构化记录 — 而非通用网页。任意 URL 请改用 novada_extract。

novada_scrape
RARE

Structured data from Amazon, Walmart, Google, TikTok, LinkedIn, GitHub, YouTube, Twitter/X, and more platforms. Use when you need e-commerce products, social posts, or job listings — NOT general websites (use extract for those). 从 Amazon、Walmart、Google、TikTok、LinkedIn、GitHub、YouTube、Twitter/X 等平台获取结构化数据。需要电商商品、社交帖子或职位列表时使用 — 通用网站请用 extract。

Use when you need typed records from a known platform. 使用场景：需要来自已知平台的结构化记录。
▸ CRAWL & MAP ×2

Crawl & Map爬取与站点地图

Explore entire sites rather than a single page. novada_map discovers all reachable URLs; novada_crawl fetches and returns the content at each URL with configurable depth and scope. 探索整个网站而非单页。novada_map 发现所有可达 URL；novada_crawl 以可配置的深度和范围抓取并返回每个 URL 的内容。

novada_crawl
EPIC

Read content from multiple pages on one site (up to 20). Use when you need an entire section of a website. Optionally run map first to discover target URLs. 读取一个站点多个页面的内容（最多 20 页）。需要整块站点内容时使用。可先运行 map 发现目标 URL。

Use when you need content from many pages of one site. 使用场景：需要一个站点多页的内容。
novada_map
COMMON

List all URLs on a website via sitemap or crawl. Use when you need to find the right page before crawl/extract. Returns URLs only, no content. Fast site reconnaissance. 通过 sitemap 或爬取列出站点上所有 URL。在 crawl/extract 前需要找到正确页面时使用。仅返回 URL，不含内容。快速站点侦察。

Use when you need the URL list before scraping. 使用场景：抓取前需要先拿到 URL 列表。
▸ RESEARCH & MONITOR ×4

Research & Monitor研究与监测

Higher-level intelligence: autonomous multi-source research, tool discovery, AI brand-mention monitoring, and page change detection. 高级智能工具：自主多源研究、工具发现、AI 品牌提及监测及页面变更检测。

novada_research
LEGENDARY

Deep multi-source research: searches multiple angles, reads top sources, returns a synthesized report with citations. Use when you need comprehensive analysis of an open-ended topic. Slower than a single search. 多源深度研究：从多个角度搜索、读取顶部来源、返回带引用的综合报告。需要对开放性主题做全面分析时使用。比单次搜索慢。

Use when a question needs synthesis across many sources. 使用场景：问题需要跨多个来源综合。
novada_discover
COMMON

List all available Novada tools grouped by category. Use when an agent wants to see the full toolset it can call before choosing one. 按类别列出所有可用 Novada 工具。Agent 在选择工具前想查看完整可调用工具集时使用。

Use when you want the full list of callable tools. 使用场景：想查看完整可调用工具清单。
novada_ai_monitor
EPIC

Search AI-company domains (chatgpt.com, perplexity.ai, anthropic.com, …) and the web for public mentions & sentiment of a brand. Searches indexed public pages — it does not query the live models. 搜索 AI 公司域名(chatgpt.com、perplexity.ai、anthropic.com 等)及网络上对某品牌的公开提及与情感。搜索已索引的公开页面 — 不查询实时模型。

Use when you track public brand mentions across AI-company sites. 使用场景：追踪品牌在 AI 公司站点上的公开提及。
novada_monitor
RARE

Track changes on a web page over time. Compares content hash and field-level diffs — useful for price tracking, news alerts, or compliance monitoring. 跟踪网页随时间的变化。比较内容哈希与字段级差异 — 适合价格追踪、新闻提醒或合规监控。

Use when you watch a page for changes over time. 使用场景：需要持续监控某页面的变化。
▸ ACCOUNT & CONFIG ×2

Account & Config账户与配置

Check your setup and manage your account. novada_setup is auth-free and uses no quota; novada_account is your single billing & wallet dashboard. 检查配置并管理账户。novada_setup 免鉴权且不消耗配额；novada_account 是你的统一账单与钱包面板。

novada_setup
COMMON

Check your API key and environment configuration. Use when you want to verify setup or see which Novada products are active. Auth-free, no quota used. 检查你的 API 密钥与环境配置。想验证配置或查看哪些 Novada 产品已激活时使用。免鉴权，不消耗配额。

Use when you want to confirm your key & active products. 使用场景：想确认密钥与已激活产品。
novada_account
RARE

Single-call account & billing dashboard. section='summary' gives wallet balance, plan quotas, recent capture logs, and health; other sections cover balance, usage history, per-product plans, and daily traffic. 一次调用的账户与账单面板。section='summary' 返回钱包余额、套餐配额、最近抓取日志与健康状态；其他 section 覆盖余额、用量历史、各产品套餐与每日流量。

Use when you need balance, quota, or usage history. 使用场景：需要余额、配额或用量历史。
▸ BROWSER ×1

Browser浏览器工具

Drive a real browser when a fetch isn't enough. novada_browser runs on the hosted endpoint via cloud CDP. Multi-step persistent flows (novada_browser_flow) run only under the local MCP — see the callout below. 当单纯抓取不够时驱动真实浏览器。novada_browser 通过云端 CDP 在托管端运行。多步持久流程(novada_browser_flow)仅在本地 MCP 下运行 — 详见下方提示框。

novada_browser
HOSTED

Automate Novada's cloud browser via CDP — navigate, click, type, screenshot, snapshot. One-shot tasks per call. Credentials auto-provisioned from your API key. Runs on the hosted endpoint. 通过 CDP 自动化 Novada 云端浏览器 — 导航、点击、输入、截图、快照。每次调用为一次性任务。凭证由你的 API 密钥自动分配。在托管端运行。

Use when you need clicks/forms, not just a fetch. 使用场景：需要点击/表单，而非单纯抓取。
⚠️

NEED MULTI-STEP PERSISTENT FLOWS? INSTALL THE LOCAL MCP. 需要多步持久流程？请安装本地 MCP。

novada_browser runs on the hosted endpoint. Only novada_browser_flow — which holds a persistent session across calls — returns 501 NOT_AVAILABLE_ON_HOSTED with an agent_instruction pointing here, because a per-request serverless isolate can't keep a session alive. novada_browser 在托管端运行。仅 novada_browser_flow(跨调用保持会话)返回 501 NOT_AVAILABLE_ON_HOSTED 并附带 agent_instruction 指引到本地 — 因为按请求隔离的 serverless 实例无法保持会话存活。

▸ PROXIES ×3

Proxies代理工具

novada_proxy returns credentials for your own HTTP client — pick a type (residential / isp / datacenter / mobile / static / dedicated). Manage proxy sub-accounts with novada_proxy_account_list and novada_proxy_account_create. 195+ countries, 100M+ residential IPs. novada_proxy 为你自己的 HTTP 客户端返回凭证 — 选择类型(residential / isp / datacenter / mobile / static / dedicated)。用 novada_proxy_account_list 与 novada_proxy_account_create 管理代理子账户。覆盖 195+ 国家，1 亿+ 住宅 IP。

novada_proxy
LEGENDARY

Proxy credentials for your own HTTP clients. type=residential|isp|datacenter|mobile|static|dedicated (default residential). Not needed for extract/crawl — those handle proxies internally. 为你自己的 HTTP 客户端返回代理凭证。type=residential|isp|datacenter|mobile|static|dedicated(默认 residential)。extract/crawl 无需此工具 — 它们内部已处理代理。

Use when your own client needs proxy user:pass:endpoint. 使用场景：你自己的客户端需要代理 user:pass:endpoint。
novada_proxy_account_list
COMMON

List proxy sub-accounts for a product (paginated). Use to audit or look up sub-account names before rotating credentials. 列出某产品的代理子账户(分页)。用于审计或在轮换凭证前查找子账户名称。

Use when you need to see existing proxy sub-accounts. 使用场景：需要查看已有代理子账户。
novada_proxy_account_create
WRITE

Create a proxy sub-account. Two-step confirm gate — pass confirm:true after human approval; without it, returns a dry-run preview instead of calling the API. 创建代理子账户。两步确认门 — 人工批准后传 confirm:true;不传则返回预览而非调用 API。

Use when you're provisioning a new proxy sub-account. 使用场景：需要开通新的代理子账户。
