▸ SEARCH & SERP ×2

Search & SERP搜索与 SERP

Query the web and retrieve structured search engine results. Use search for general content retrieval; use serp when you need raw, structured SERP JSON for analysis or ranking tasks. 查询网络内容或获取结构化搜索引擎结果。通用内容获取用 search；需要原始结构化 SERP JSON 用于分析或排名任务时使用 serp。

search
RARE

Web search across major engines. Returns clean text results with titles, URLs, and snippets. The primary tool for any question-answering or research task that starts from the open web. 跨主流引擎的网页搜索。返回带标题、URL 和摘要的纯文本结果。适合从公开网络出发的问答或研究任务。

Use when you need to find relevant pages from the open web. 使用场景：需要从公开网络找到相关页面。
serp
RARE

Structured Google / Bing SERP JSON. Returns organic results, ads, featured snippets, knowledge panels, and related searches as typed objects — ideal for SEO analysis and competitive intelligence. 结构化的 Google / Bing SERP JSON。返回自然结果、广告、精选摘要、知识面板和相关搜索的类型化对象 — 适合 SEO 分析和竞争情报。

Use when you need raw SERP JSON for SEO or ranking analysis. 使用场景：需要原始 SERP JSON 做 SEO 或排名分析。
▸ SCRAPE & EXTRACT ×3

Scrape & Extract抓取与提取

Turn any URL into structured data. scrape gives clean Markdown prose; extract applies a schema to produce typed JSON fields; unblock handles JS-heavy or bot-protected pages by returning the fully rendered HTML. 将任意 URL 转化为结构化数据。scrape 输出干净的 Markdown 正文；extract 按 schema 生成类型化 JSON 字段；unblock 处理 JS 渲染或有反爬保护的页面并返回完整 HTML。

scrape
RARE

Single-URL clean Markdown content. Strips nav, ads, and boilerplate. Returns the main body in Markdown, ready to pass directly to an LLM context window. 单 URL 干净 Markdown 正文。去除导航、广告与无关内容，将主体内容以 Markdown 返回，可直接写入 LLM 上下文窗口。

Use when you want readable Markdown from one known URL. 使用场景：想从单个已知 URL 拿到可读 Markdown。
extract
EPIC

Schema-driven structured JSON extraction. Define the fields you want (e.g. price, title, date, author) and get back a typed JSON object — no XPath or CSS selectors needed. 按 schema 提取结构化 JSON。定义所需字段（如价格、标题、日期、作者），返回类型化 JSON 对象 — 无需 XPath 或 CSS 选择器。

Use when you need specific typed fields, not the whole page. 使用场景：只要特定类型化字段，而非整页内容。
unblock
EPIC

Raw rendered HTML of blocked or JS-heavy pages. Bypasses common bot protection to return the fully rendered DOM — use when scrape returns incomplete content. 返回被封锁或 JS 渲染页面的完整 HTML。绕过常见反爬机制并返回完整渲染的 DOM — 当 scrape 返回内容不完整时使用。

Use when scrape comes back empty on a protected page. 使用场景：scrape 在受保护页面返回空内容时。
▸ CRAWL & MAP ×2

Crawl & Map爬取与站点地图

Explore entire sites rather than a single page. map discovers all reachable URLs; crawl fetches and returns the content at each URL with configurable depth and scope. 探索整个网站而非单页。map 发现所有可达 URL；crawl 以可配置的深度和范围抓取并返回每个 URL 的内容。

crawl
EPIC

Multi-page crawl with depth control. Follows internal links from a seed URL and returns clean Markdown for each page up to the configured depth and page limit. Respects robots.txt by default. 可控深度的多页面爬取。从种子 URL 跟踪内部链接，并在配置的深度与页数限制内返回每页的干净 Markdown。默认遵守 robots.txt。

Use when you need content from many pages of one site. 使用场景：需要一个站点多页的内容。
map
COMMON

Discover all URLs on a site. Returns a flat list of reachable URLs from the given root — useful as a first step before targeted scraping or to audit site structure. 发现站点上所有 URL。从给定根 URL 返回可达 URL 的平铺列表 — 适合定向抓取前的前置步骤或站点结构审计。

Use when you need the URL list before scraping. 使用场景：抓取前需要先拿到 URL 列表。
▸ RESEARCH & VERIFY ×5

Research & Verify研究与验证

Higher-level intelligence tools: autonomous multi-source research, fact-checking with confidence scoring, source discovery, AI brand monitoring, and real-time page change detection. 高级智能工具：自主多源研究、带置信度评分的事实核查、来源发现、AI 品牌监测及实时页面变更检测。

research
LEGENDARY

Deep multi-source research agent. Searches, fetches, and synthesizes information from multiple sources into a coherent answer with citations — equivalent to a junior analyst pass over the web. 多源深度研究 Agent。搜索、抓取并综合多个来源的信息，输出带引用的完整答案 — 相当于初级分析师的网络检索汇总。

Use when a question needs synthesis across many sources. 使用场景：问题需要跨多个来源综合。
verify
EPIC

Source check and confidence scoring. Given a claim, searches the web for corroborating or contradicting evidence and returns a confidence score with supporting sources. 来源核验与置信度打分。针对给定声明，搜索网络中的支持或反驳证据，并返回带来源的置信度评分。

Use when you need to fact-check a claim with sources. 使用场景：需要用来源核查某个声明。
discover
RARE

Find related sources for a topic. Given a seed URL or topic, surfaces authoritative sources, similar sites, and key references — useful for building a corpus or competitive research. 围绕主题发现相关来源。给定种子 URL 或主题，挖掘权威来源、相似网站和关键参考 — 适合构建语料库或竞争调研。

Use when you're building a source list around a topic. 使用场景：围绕主题搭建来源清单。
ai_monitor
EPIC

Monitor how AI models reference a brand. Tracks mentions, sentiment, and positioning of a brand or product across AI-generated content and model outputs over time. 监测 AI 模型如何引用品牌。追踪品牌或产品在 AI 生成内容和模型输出中的提及、情感倾向和定位变化。

Use when you track brand presence inside AI outputs. 使用场景：追踪品牌在 AI 输出中的表现。
monitor
RARE

Detect page changes over time. Periodically checks a URL for content changes and surfaces diffs — useful for price tracking, news alerts, or compliance monitoring. 检测页面随时间的变化。定期检查 URL 内容并输出差异 — 适合价格追踪、新闻提醒或合规监控。

Use when you watch a page for changes over time. 使用场景：需要持续监控某页面的变化。
▸ SCRAPER API — ASYNC ×3

Scraper APIScraper API

Task-based async structured scraping of 129+ platforms (Amazon, LinkedIn, Google Maps, …). Submit a job, poll for status, retrieve the result — designed for high-volume or slow-rendering targets that would time out in a synchronous call. 基于任务的异步结构化抓取，支持 129+ 平台（亚马逊、LinkedIn、谷歌地图等）。提交任务 → 轮询状态 → 获取结果 — 专为高并发或渲染慢的目标设计，避免同步调用超时。

scraper_submit
EPIC · ASYNC

Submit an async scraper task. Specify the target platform and parameters; returns a task_id immediately. The task runs in the background across Novada's scraper infrastructure. 提交异步抓取任务。指定目标平台与参数，立即返回 task_id。任务在 Novada 抓取基础设施后台运行。

Step 1 — kick off a platform scrape job. 第 1 步 — 启动平台抓取任务。
scraper_status
COMMON · ASYNC

Poll a task's status by task_id. Returns pending, running, completed, or failed. Call this before retrieving results to avoid wasted quota. 通过 task_id 轮询任务状态。返回 pending、running、completed 或 failed。取结果前先调用以避免消耗额度。

Step 2 — poll until the job completes. 第 2 步 — 轮询直至任务完成。
scraper_result
EPIC · ASYNC

Retrieve the completed result for a task. Returns the structured JSON payload from the scraper — platform-specific schema (product data, listing data, review data, etc.). 获取已完成任务的结果。返回抓取器输出的结构化 JSON — 根据平台有特定 schema（商品数据、列表数据、评论数据等）。

Step 3 — pull the structured payload. 第 3 步 — 取出结构化结果。
▸ PROXIES ×7

Proxies代理工具

One key unlocks six distinct proxy products plus a universal router. All tools share the same parameter surface — pass a URL and optional country/session parameters; get back the proxied response. 195+ countries, 100M+ residential IPs, sub-second rotation. 一把密钥解锁六种代理产品，外加一个通用路由器。所有工具参数一致 — 传入 URL 及可选的国家/会话参数，返回代理响应。覆盖 195+ 国家，1 亿+ 住宅 IP，毫秒级轮换。

novada_proxy
LEGENDARY

Universal proxy router. Automatically selects the best proxy type for the target URL — the recommended default for agents that don't need to specify a product explicitly. 通用代理路由器。自动为目标 URL 选择最佳代理类型 — 适合无需显式指定产品的 Agent 的推荐默认选项。

Use when you want the router to pick the proxy for you. 使用场景：让路由器自动为你选择代理。
novada_proxy_residential
EPIC

Residential proxies. Routes requests through real consumer IP addresses. Highest trust score for geo-sensitive or heavily protected targets. 100M+ IPs across 195+ countries. 住宅代理。通过真实消费者 IP 路由请求。对地理敏感或重度保护目标信任度最高。195+ 国家，1 亿+ IP。

Use when the target heavily blocks datacenter IPs. 使用场景：目标重度封锁机房 IP。
novada_proxy_datacenter
COMMON

Datacenter proxies. Fast, high-throughput proxies from commercial data centers. Best for speed-sensitive bulk crawls where residential trust level is not required. 机房代理。来自商业数据中心的高速、高吞吐代理。适合不需要住宅级信任、对速度敏感的批量爬取。

Use when speed matters more than trust score. 使用场景：速度优先于信任度。
novada_proxy_isp
RARE

ISP proxies. Static IPs assigned by real ISPs — combines the trust profile of residential IPs with the stability and speed of datacenter infrastructure. ISP 代理。由真实 ISP 分配的静态 IP — 兼具住宅 IP 的信任度与机房基础设施的稳定性和速度。

Use when you need residential trust with datacenter speed. 使用场景：既要住宅信任度又要机房速度。
novada_proxy_dedicated
RARE

Dedicated datacenter proxies. Exclusive IPs not shared with other users. Ideal for targets that blacklist shared IPs or for workflows requiring consistent session identity. 独享机房代理。不与其他用户共享的专用 IP。适合将共享 IP 列入黑名单的目标，或需要一致会话身份的工作流。

Use when you need an IP no one else touches. 使用场景：需要不与他人共享的独享 IP。
novada_proxy_static
RARE

Static residential proxies. Long-lived residential IPs that persist across requests — use when a site tracks session continuity or requires login state to be maintained. 静态住宅代理。跨请求持久存在的长效住宅 IP — 适用于追踪会话连续性或需要保持登录状态的网站。

Use when a site must see the same IP across requests. 使用场景：网站需要跨请求看到同一 IP。
novada_proxy_mobile
EPIC

Mobile carrier proxies. Routes through 4G/5G mobile network IPs. Highest trust for mobile-first platforms and apps that differentiate between mobile and desktop traffic. 移动运营商代理。通过 4G/5G 移动网络 IP 路由。对区分移动端与桌面端流量的移动优先平台和应用信任度最高。

Use when the target treats mobile traffic differently. 使用场景：目标对移动流量区别对待。

▸ BROWSER — LOCAL MCP ONLY ×2

Browser浏览器工具

Playwright-powered headless browser and flow automation. These tools require the local MCP runtime and return 501 NOT_AVAILABLE_ON_HOSTED on the cloud endpoint — see the callout below. 基于 Playwright 的无头浏览器和流程自动化工具。这些工具需要本地 MCP 运行时，在云端点会返回 501 NOT_AVAILABLE_ON_HOSTED — 详见下方提示框。

novada_browser
LOCAL ONLY

Headless browser automation. Navigate URLs, click elements, fill forms, take screenshots, and return the rendered DOM — powered by Playwright. Use when scrape or unblock can't handle the interaction flow. 无头浏览器自动化。导航 URL、点击元素、填写表单、截图并返回渲染后的 DOM — 基于 Playwright。当 scrape 或 unblock 无法处理交互流程时使用。

Use when you need clicks/forms, not just a fetch. 使用场景：需要点击/表单，而非单纯抓取。
novada_browser_flow
LOCAL ONLY

Multi-step browser flow automation. Define a sequence of browser actions (navigate → fill → click → wait → extract) and execute them atomically. Useful for login-gated scrapes or checkout flows. 多步骤浏览器流程自动化。定义动作序列（导航 → 填写 → 点击 → 等待 → 提取）并原子化执行。适合需要登录的抓取或结账流程。

Use when a login-gated flow needs multiple steps. 使用场景：需登录的流程需要多步骤。
