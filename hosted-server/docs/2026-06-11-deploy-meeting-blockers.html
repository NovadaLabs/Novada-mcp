🛑 阻塞 #1 — Route 53 CNAME(DNS)

问题

客户连接的是 https://mcp.novada.com/mcp,但这个子域名还没解析。需要在 novada.com 的 Route 53 hosted zone 添加一条 CNAME 记录:

Name:  mcp
Type:  CNAME
Value: cname.vercel-dns.com
TTL:   300

谁能解决

需要:novada.com Route 53 hosted zone 的 AWS 控制台访问权限

已确认 novada.com 的 DNS 在 AWS Route 53(dig NS novada.com 返回 *.awsdns-*.org / .net / .com / .co.uk 四个 NS),但 不在 tongwu 的 contact@prismma.ai 个人 AWS 账户里(那个账户 Hosted Zones 数量 = 0)。

所以这条 CNAME 必须由 持有公司主 AWS 账户(novada.com hosted zone 所在账户)的人来加。最可能是 fudong / 公司域内的运维同事。

风险评估

  • 新增子域,零风险:只是给 novada.com 加一条新的 CNAME 子记录,不修改任何现有记录(www / api / mail 等),不动 NS / SOA,完全不会影响 novada.com 主站或邮箱。
  • 可回滚:1 秒内删除,DNS 在 TTL 300 秒内全球回退。
  • 已多次实操:类似操作公司其他子域名(api.novada.com 等)早已存在。

给 fudong 的具体步骤(<2 分钟)

  1. 登录 AWS 主账户 → Route 53 → Hosted zones → 选 novada.com
  2. 点 Create record
  3. 填:Name = mcp,Type = CNAME,Value = cname.vercel-dns.com,TTL = 300
  4. Save。完。

⚠️ Vercel CLI 提示如果 CNAME 因任何原因不通,可以改用 A 记录:Type A,Value 76.76.21.21。两种方式都行,CNAME 更"标准"。

🛑 阻塞 #2 — Vercel KV 存储(决定方案 + 计费)

问题

MCP server 用 KV 来做两件事:(a) 按 token 的月度调用配额(free 5000/月)、(b) 按 IP 的每分钟速率限制(默认 60 req/min)。这两个值都必须存在某个共享 store 里(serverless 各个实例之间)。

目前 /api/mcp 返回 FUNCTION_INVOCATION_FAILED,根因是 @vercel/kv 模块在缺少 KV_REST_API_URL / KV_REST_API_TOKEN 时 import 阶段就崩溃。

今天发现:Vercel 把 Upstash for Redis 的 free tier 取消了,新建一个独立的 Upstash KV 起步价 $8/月。

三种方案,要选一种

方案 A — 复用现有 prismma-redis (Upstash Free)推荐 $0 / 月
原理:团队里 prismma-redis 已经在 Upstash Free 上跑,目前只连了 prismma-ai。MCP server 用同一个 Redis 实例,但所有 key 加 novada: 前缀,与 Prismma 完全隔离。
需要做的:(1) vercel env pull 从 prismma-ai 拿 KV 凭据 → (2) vercel env add 到 novada-mcpserver → (3) 改 10 行代码加 key prefix → (4) redeploy。约 10 分钟。
风险:共享一个 Upstash 实例 — 如果 Prismma 流量爆发,Novada 也会跟着被限速。v1 演示完全够用;真生产再分。
需要 yixuan / Prismma 同意:是,因为这是改 Prismma 的基础设施。
方案 B — 给 novada-mcpserver 单独开 KV $8 / 月起
原理:Vercel Storage → 新开一个 Upstash for Redis(或官方 Redis),最低 250MB / $8 月。
需要做的:tongwu 在 Vercel dashboard 点 Create → 选 plan → 信用卡确认。约 3 分钟。
优点:独立资源,与 Prismma 解耦。
缺点:$8/月持续支出,Novada 还没收入。
需要 Ethan 同意计费:是,如果走公司账。
方案 C — 改代码,让 MCP 在没有 KV 时优雅降级 $0,但功能降级
原理:把 @vercel/kv 改为 lazy import,KV 缺失时直接跳过配额/速率检查,工具调用正常返回。
需要做的:tongwu 改 ~30 行代码 + 重新部署。约 20 分钟。
缺点:(1) 无配额跟踪 — 任何人拿到 token 可以无限调用;(2) 无速率限制 — 容易被滥用;(3) 违反 PROJECT_STATE.md §6 的 critical patch #4("fail loud, no silent degrade")。
建议:只作为应急。如果 A 和 B 都临时拍不下来,先用 C 撑过明天的 demo,后面补 A 或 B。
