What shipped this update本次更新上线了什么

LAYER A  Account & billing tools — now on hosted账户与计费工具 — 已上线托管端

8 account tools are now callable on mcp.novada.com: wallet_balance, wallet_usage_record, plan_balance_all, proxy_account_list, proxy_account_create (write, confirm-gated), traffic_daily, account_summary, capture_logs. An agent can check its balance, audit traffic, and provision a proxy account — without the user leaving the chat. mcp.novada.com 现已可调用 8 个账户工具:wallet_balance、wallet_usage_record、plan_balance_all、proxy_account_list、proxy_account_create(写操作,二次确认)、traffic_daily、account_summary、capture_logs。Agent 可查余额、审计流量、开通代理子账户 —— 用户无需离开对话。

🔒 Security fix (gating)安全修复(前置): the tools previously fell back to the operator's key — on pass-through hosting that would have leaked the operator's wallet to every customer. Fixed: the customer's own key is threaded through all 8. Verified — bogus key → 401 (no fallback), real key → correct balance. 这些工具此前会回退到运营者的密钥 —— 在透传托管下会把运营者钱包泄露给每个客户。已修复:8 个工具全部改用客户自己的密钥。已验证 —— 错误密钥 → 401(不回退),正确密钥 → 返回正确余额。

LAYER B  Slim-endpoint filter ?tools= / ?groups=精简端点过滤 ?tools= / ?groups=

Matches BrightData's ?groups= pattern. mcp.novada.com/mcp?groups=search,scrape serves only those tools — fewer tools = less agent context overhead and higher accuracy. 8 groups: search, scrape, crawl, research, scraper, proxy, account, browser. 对齐 BrightData 的 ?groups= 模式。mcp.novada.com/mcp?groups=search,scrape 只暴露这些工具 —— 工具越少,Agent 上下文负担越小、准确率越高。8 个分组:search、scrape、crawl、research、scraper、proxy、account、browser。

PAGE  /configure — tool picker → custom URL/configure — 工具配置器 → 自定义 URL

Novada clone of BrightData's "Configure MCP": per-category / per-tool picker that live-generates a scoped endpoint URL (now backed by the real Layer B filter). Own key, no demo token.BrightData "Configure MCP" 的 Novada 克隆:按分类 / 按工具勾选,实时生成精简端点 URL(由真实的 Layer B 过滤支撑)。客户自有密钥,无演示 token。
